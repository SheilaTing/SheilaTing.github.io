============================================

serif：衬线字体，就是边角有特殊修饰的字体。宋体字就是有衬线字体。Serif的意思是，在字的笔划开始及结束的地方有额外的装饰，而且笔划的粗细会因直横的不同而有不同。

sans-serif：无衬线字体，无特殊修饰的字体。sans是法语，意思是“没有”。黑体字就是无衬线字体。Sans Serif则没有这些额外的装饰，笔划粗细大致差不多。

monospace：等宽字体，每个字母都一般宽的字体

css 布局案例 数字键盘

这个有点经典哎 我要布局数字键盘，OK键比其他数字键都大一些，如果使用 flex布局，ok键会把同行的数字键盘的高度一起拉高，造成不协调，那么我们使用浮动，浮动的过程中，所有键左浮动，碰到 ok 时发现它依然把同行的键高度拉高了，这次是因为浮动元素彼此之间可以感知到对方，依次在上一个浮动元素的后面摆放，下一行的元素感知到 ok 存在，它的高度又高，所以会造成下一行在ok的下面显示，中间与上一层不是ok的键形成空白。此时如何解决呢？此时我们可以把ok键 再次设置浮动方向为右侧浮动，这样它后面的元素还是左侧浮动，它们回去找前面左浮动元素跟在它们后面，我们把每个数字键的宽度设置好后，ok键会准确的落在界面最右侧，其他键放不下，自然会另起一行，形成键盘的布局，nice；

tips 一般布局时给元素宽高时会按照设计稿的尺寸来给，可以写精确的 px,也可以写该元素在整个设计稿中占据的宽高的比例，然后设置相应的 vh 和 vw,这样的好处是适配好，在不同的手机屏幕上，用户看到的元素的比例是一致的

vue中 组件间传值可以通过 props 属性 如果是驼峰写法，传递时要写成 a-b ,其实 css 也可以利用这个方法来传递，我们把传递来的值绑定到一个元素的 class 上，然后在传值的地方给这个class 加样式，就实现了，样式的跨组件传递，当然要记得，style 不要写成 scope 的否则，只会在当前组件中生效。这样写的缺点是，如果需要控制的样式太多，那么我们就要为了样式传递很多的值，那有没有更好的办法呢？ 阿里的 antd 团队，发明了一个办法，就是在传递时，我们不直接传递这个样式的名字，而是只传递一个前缀过去，传过去后，把这个前缀绑定在需要控制样式的元素上再加上它本身的class，这样我们只需要传递一个值过去就能控制多个元素，感觉这样写跟写全局的没差别呀，最终还是要拿到这个class 的全称来设置样式啊！。。。。。 无语，饶了一圈

模块化好处

把一个功能或者一个页面拆分为不同的组件文件，那么每一个文件内的代码量就会减少，虽然增加了，串联起来的复杂度，但代码量少就意味着出错的几率小，而且就算一个组件除了错，不会影响其他组件；

vue 控制 css 是否显示的方式

对象的方式，写好的 css 样式名称，通过条件指定它的 true or false，或者直接通过条件判断是否给这个元素这个样式；

slice（起始下标，结束下标（不包含，支持负数，-1代表最后一位，-2倒数第二位））这在删除最后一位时非常有用，我们可以将字符串变为数组后 pop，也可以直接截取到最后一位即可；

流程总结

先搭建环境，配置一些全局css,然后写 静态页面，然后写页面的同时如果发现组件代码超过了200行， 建议拆分组件内容到不同的文件中，减少bug，然后编写 js 代码，无可避免要使用到组件间的传值。组件中的js也就是一些实现点击时的一些功能，和一些初始数据的渲染以及css的控制，由于我们把一个页面的组件拆分成了多个组件，那么我们再最终实现功能需要这些组件间的数据的相互配合时，我们就需要把这些数据整合起来，也就是从子组件中把这个数据传递到父组件中来，可以使用 $emit 来触发事件，父组件中监听即可，然后父组件拿到数据后再做统一的处理；例如把这些数据组装成一个对象

vue .sync

当父组件出现 ：value =”值” v-on:update:value=”…”可以使用 .sync 省略掉绑定的事件，变成 ：value.sync=’值A’ 当然你在触发 $this.emit(‘update:value’,值)，传递的新的值，是你想要改变父组件中传递给你的这个变量的值，有点迷糊哎，还是一步一步写清楚吧，嘻嘻

思路哈

一般情况下，很多组件的默认值是从外界传递过来的，我们无需再组件内部自己构造，我们只需接收外部传递过来的数据，然后在组件内部根据用户的操作，获取到操作后的值，然后传递给父组件（传给我们初始值的组件），由它统一改变即可。这样子组件不需要再维护自己的数据，无疑变得十分简单了。
